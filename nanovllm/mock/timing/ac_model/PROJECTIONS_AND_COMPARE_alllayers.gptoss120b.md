# GPT-OSS Full-Layer Projection and InferenceX Comparison

This report uses Nsys profile to aggregate a full model worth of clean layer slices per config: 18 causal layers and 18 sliding-window layers. Times are from CUPTI GPU kernel timestamps on the selected device lane.

The local profiles are ISL sweep profiles with OSL=128. The InferenceX CSV comparison rows are ISL=8192, OSL=1024, so the comparison is a useful sanity check but not an exact methodology match.

Some profile windows contain a cudaGraph/capture boundary inside the adjacent raw layer stream. To avoid counting boundary-spanning pseudo-layers, this report uses only the modal 12-kernel or 13-kernel layer shape for each config and selects the middle 18 clean causal plus middle 18 clean sliding layers.

Overlap-adjusted tables subtract only the union of overlapped timestamp intervals inside the selected clean layers. They do not add serving-loop, scheduler, sampler, request, or benchmark-client overhead.

The main trend is that BMM overlap is nearly flat in absolute time, usually around 0.14-0.17 ms, so its percentage falls as concurrency increases and the full-layer kernel sum grows. All-reduce overlap is much more TP-sensitive: TP2 is around 0.18-0.21 ms, TP4 is around 0.21-0.35 ms, and TP8 is around 0.46-0.85 ms in this profile set. At TP8 the overlap adjustment is dominated by communication hiding; at TP2 the communication and BMM terms are closer in size.
